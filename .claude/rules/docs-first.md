# Documentation-first rules

- Before coding, read local docs relevant to the task.
- Prefer repository-local examples and knowledge packs over web research.
- If a new reusable pattern appears, add it to `docs/agent/features/` or `docs/snippets/`.
- When implementing DB work, read `docs/snippets/db-migrations.md`.
- When implementing parsing work, read `docs/snippets/parser.md`.
- When implementing roadmap work, align with `docs/agent/backend-roadmap.md`.
