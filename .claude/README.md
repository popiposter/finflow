# .claude notes

Use `.claude/rules` for small, focused, low-duplication rules.

## Guidelines

- Each rule file should cover one narrow topic.
- Do not copy large chunks of process guidance into multiple rule files.
- When a rule is general across repositories, prefer the external playbook repo.
- When a rule is specific to FinFlow testing or CI, prefer `docs/testing-architecture.md` and keep the rule file minimal.
